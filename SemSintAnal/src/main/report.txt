извлечение Ngramm
алгоритм:
1. лемматизируем текст
2. извлекаем все н-граммы заданной длины, добавляя их в hashmap в качестве ключа
    значение для этого ключа - индексы вхождений этой н-граммы. Тогда если она повторяется
    нам не нужно заново находить индексы для поиска максимального по вхождению расширения.
    вдобавок размер этого массива это частота встречаемости в корпусе.
3. для каждой уникальной найденной н-граммы - находим максмальные расширения и проверяем
   критерий устойчивости. Для улучшения производительности это делается параллельно. Мы
   можем себе это позволить, потому что поиск максимального расширения предполагает работу
   с единственным разделяемым ресурсом - лемматизированным текстом, но при это доступ к нему
   происходит только на запись, поэтому raise condition не случается.
4. Опциональный шаг, сортируем устойчивые нграммы по встречаемости в корпусе
